下流工程から上流工程、そして保守工程を経験してきました。
そのため、開発メンバや保守メンバの事を考えた設計や資料作成および開発を行うように心がけています。
顧客との調整および開発メンバとのコミュニケーションの両方を重要視しています。
また、顧客からの信頼により、指名されて案件や作業に加わったこともあります。
新しい技術であっても、自分で調査・検証および有識者へ積極的に相談することで乗り越えてきました。
その結果、社内初のAWSを利用した基盤の設計と構築を行い、社内でも高く評価されました。
今後も上流工程にも下流工程にも積極的に関わり、顧客折衝能力と技術の両方を学んでいきたいと考えています。

ABOUT

　佐藤美生（サトウミウ）
　1987年7月8日生まれ、31歳（2019/4/29現在）
　北海道出身、在住。
　音楽鑑賞、ソーシャルゲーム、旅行、写真などが好き（ヘッダー写真は自作）
　学生時代は吹奏楽部でフルートを吹いており、就職して神奈川にいた頃は月1でレッスンに通っていました。

　2009年、新卒入社。
　2014年、業務の過労とキャリアパスの不安により、双極性障害を発症し半年休職。
　2016年、会社への不信感などで病状が悪化し再度休職。そのまま退職。
　その後、東京で就職と退職を繰り返した後、2018年から北海道の実家にてリモートワークを開始。

　子供の頃から物作りが好きなので、開発業務に就いたのは必然だったと感じています。作ったもので、顧客に喜んで貰えるような、顧客にとってプラスアルファになるような仕事がしたいと思っています。また、エンドユーザが近いWebサービスに興味を持っています。
　現状ゼロから物作りを考える事は苦手であることがウィークポイントであると認識していますが、1あればそこから発展させることは出来ると考えています。
　今まで世に無かった新しいサービスやシステムを考えて作りエンドユーザに使って貰う事を、エンジニアとしての目標としています。

　在学中に、基本情報技術者、ソフトウェア開発技術者を取得。2013年、情報セキュリティスペシャリスト取得。

SKILL

JavaScript

Java

学生時代に授業でやっただけで、実務経験は少ししかありません。SpringMVCの使用経験があります。

PHP

一番得意な言語ですが、フレームワークはあまり経験がありません。CakePHPの使用経験が少しあります。

Oracle

サーバ構築も行ったことがあります。DBはOracleの使用が多かったですが、MySQL、PostgreSQLの使用経験もあります。

Linux/UNIX

得意分野です。基盤保守・運用時代、Linuxサーバを10台ほど担当していました。主なOSはRedhat、CentOSです。構築から運用まで一通り出来ます。

bash

得意分野です。難しいシェルスクリプトも記述可能です。

AWS

構築から運用まで行った事があります。EC2、RDS、S3、Route 53、SESなど使用したことがあります。社内初の構築実績となり、社内顧客共に高評価を頂きました。

要件定義/設計

サブリーダーとして、要件定義及び画面設計に携わり、顧客折衝を行いました。顧客からは対応について高評価を頂きました。開発及び保守運用工程を経験してきたため、開発メンバや保守メンバのことを考えた設計や資料作成を心がけています。

保守/運用

大手コンビニエンスストアのマスタ管理サブの基盤保守を一人で行っていました。サーバ台数は10台程、障害発生時の切り分け、対応などを行いました。Apache、OracleApplicationServer、Tomcatの経験ありです。
